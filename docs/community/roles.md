# Community Roles (Privacy-First)

## Users

- Install and use ShadowNet.
- Import bundles from trusted publishers.
- Share feedback only when safe.

## Testers

- Run staged scenarios from the field testing plan.
- Generate `report.json` and share manually.
- Avoid sharing sensitive logs/configs.

## Outside Helpers

- Generate signed bundles and update packages.
- Help with offline distribution (USB/Bluetooth/messaging).
- Assist with verification steps (checksums, signatures).

## Developers

- Implement fixes and improvements.
- Add tests and documentation.
- Review security implications of changes.

## Maintainers

- Own release signing and manifests.
- Decide project roadmap and safety posture.
- Review sensitive changes (crypto, networking, distribution).

## General Rules

- No user tracking or telemetry proposals.
- Assume adversarial environment when designing features.
- Prefer safe defaults and explicit failure messages.
