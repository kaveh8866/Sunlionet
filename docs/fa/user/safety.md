# راهنمای ایمنی ShadowNet

## محدودیت‌های مهم

- ShadowNet **گمنامی را تضمین نمی‌کند**.
- ShadowNet **حفاظت حقوقی ایجاد نمی‌کند**.
- ShadowNet **تمام ریسک‌های نظارت را حذف نمی‌کند**.

## پیش‌فرض‌های ایمن در این پروژه

- بعد از نصب، عملیات به‌صورت **local-only** انجام می‌شود.
- هیچ telemetry یا tracking وجود ندارد.
- وضعیت محلی پروفایل‌ها/قالب‌ها به‌صورت رمزگذاری‌شده نگهداری می‌شود.
- لاگ‌ها حداقلی و bounded هستند.

## مسئولیت‌های کاربر

- فایل‌ها را قبل از نصب/واردکردن/به‌روزرسانی **تأیید** کنید.
- کلیدهای signer مورد اعتماد را فقط از منابع تأییدشده نگه دارید.
- از اشتراک‌گذاری لاگ‌هایی که زمینه حساس شبکه دارند خودداری کنید.
- اگر ریسک شخصی بالاست، از نام‌گذاری خنثی برای اپ/سرویس استفاده کنید.

## شرایط پرریسک

- اگر تحت ریسک فوری هستید، سریع قطع ارتباط کنید.
- اگر ریسک ضبط دستگاه بالاست، داده‌های برنامه را پاک کنید و bundleهای واردشده را حذف کنید.
- نسخه‌های اضافی فایل‌های bundle/update را نگه ندارید.

## ایمنی اجتماعی

- دیگران را بدون توضیح ریسک‌ها برای نصب تحت فشار نگذارید.
- فقط فایل‌های تأییدشده را به اشتراک بگذارید و از بازنشر فایل‌های نامطمئن پرهیز کنید.
