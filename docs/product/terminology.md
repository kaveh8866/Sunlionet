# Product Terminology

Use these terms consistently across Android UI, website copy, and tests.

## Connection State

- `Disconnected`
- `Connecting`
- `Connected`
- `Error` (or `Connection failed` for detail text)

## Primary Actions

- `Connect`
- `Disconnect`
- `Import configuration`

Avoid alternatives like:

- `Import bundle` as the primary label (keep `bundle` as a technical noun in docs context only)
- `Start tunnel` / `Stop tunnel` for user-facing primary actions

## Readiness Language

- `Configuration: Required`
- `Configuration: Ready`
- `Ready to connect`

## Product Maturity Label

- `Beta (MVP-level)`

When possible in user-facing copy, include a short qualifier:

- "test build", "experimental", or "may be unstable"

## Details and Troubleshooting

- `Show advanced details` / `Hide advanced details`
- `Details`
- `Troubleshooting`

## Verification and Install

- `Verification`
- `SHA256`
- `Signature`
- `Install / run`

## Notes

- Prefer plain language first, then technical detail.
- Do not communicate status by color only; always include explicit text.
