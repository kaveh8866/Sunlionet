# سان‌لاین‌نت (SunLionet)

سان‌لاین‌نت یک سامانه «آفلاین‌محور» و مبتنی بر بسته‌های امضاشده/رمزشده است که با هدفِ حفظ حریم خصوصی و فراهم کردن ارتباطاتِ مقاوم در شبکه‌های پرریسک و محدود طراحی شده است. این پروژه از یک کدبیس مشترک دو باینری هماهنگ تولید می‌کند:

- SunLionet Inside (نسخه داخل): روی لپ‌تاپ/موبایل اجرا می‌شود، اختلال شبکه را تشخیص می‌دهد، با سیاست‌گذاری قطعی (و در صورت نیاز یک مشاور محلی محدود) پروفایل‌های sing-box را چرخش می‌دهد و در خاموشی کامل می‌تواند از Bluetooth Mesh برای تبادل محلی استفاده کند.
- SunLionet Outside (نسخه خارج/حامی): بیرون از شبکه محدود اجرا می‌شود، پروفایل‌های جدید را تولید/اعتبارسنجی می‌کند و بسته‌های امضاشده/رمزشده را از کانال‌های مورد اعتماد (مثل Signal) به Inside می‌رساند.

یادداشت مهاجرت: نام مخزن در دوره گذار هنوز `shadownet-agent` است؛ بنابراین ممکن است بعضی مسیرها، نام‌گذاری آرتیفکت‌ها، و شناسه‌های پروتکل هنوز از نام‌گذاری قدیمی `shadownet-*` استفاده کنند.

سایت مستندات (GitHub Pages): https://kaveh8866.github.io/Sunlionet/

## شروع سریع (برای توسعه)

پیش‌نیازها:

- Go نسخه 1.21 به بالا
- ویندوز: PowerShell 7 به بالا
 
پیشنهاد: از Go نسخه 1.25 به بالا استفاده کنید (مطابق `go.mod`).

اجرای تست‌ها:

- ویندوز:

  ```powershell
  .\scripts\run_tests.ps1
  ```

- لینوکس/مک:

  ```bash
  ./scripts/run_tests.sh
  ```

## بیلد

### SunLionet Inside (داخل ایران)

```bash
go build -tags inside -ldflags="-s -w" -o bin/sunlionet-inside ./cmd/inside/
```

### SunLionet Outside (خارج/حامی)

```bash
go build -tags outside -ldflags="-s -w" -o bin/sunlionet-outside ./cmd/outside/
```

## توزیع کانفیگ (خارج → داخل)

سان‌لاین‌نت عمداً API مرکزی برای دریافت کانفیگ ندارد. بذرهای اولیه عمدتاً از راه‌های زیر می‌رسند:

1. پیام‌های ناپدیدشونده Signal با بسته‌های `snb://v2:` (امضاشده و رمزگذاری‌شده)
2. QR کد برای بوت‌استرپ حضوری
3. Bluetooth Mesh (داخل ↔ داخل) برای خاموشی اینترنت
