# شَدونِت (ShadowNet) — عامل داخل ایران + عامل خارج از ایران

ShadowNet یک عامل «آفلاین‌محور» برای دورزدن سانسور و DPI است که برای شرایط پرریسک (مثل ایران) طراحی شده است. این پروژه از یک کدبیس مشترک دو باینری هماهنگ تولید می‌کند:

- ShadowNet-Inside (نسخه داخل ایران): روی لپ‌تاپ/موبایل داخل ایران اجرا می‌شود، اختلال را تشخیص می‌دهد، با سیاست‌گذاری قطعی و در صورت نیاز LLM محدود، پروفایل‌های sing-box را چرخش می‌دهد و در خاموشی کامل از Bluetooth Mesh برای تبادل محلی پروکسی استفاده می‌کند.
- ShadowNet-Outside (نسخه خارج از ایران): بیرون از ایران اجرا می‌شود، «کارخانه کانفیگ» است، پروفایل‌های جدید را تولید/آزمایش می‌کند و بسته‌های امضاشده/رمزشده را عمدتاً از طریق Signal به Inside ارسال می‌کند.

سایت مستندات (GitHub Pages): https://kaveh8866.github.io/shadownet-agent/ (پس از فعال‌سازی Pages و تنظیم منبع روی `/docs` در شاخه `main`).

## شروع سریع (برای توسعه)

پیش‌نیازها:

- Go نسخه 1.21 به بالا
- ویندوز: PowerShell 7 به بالا

اجرای تست‌ها:

- ویندوز:

  ```powershell
  .\scripts\run_tests.ps1
  ```

- لینوکس/مک:

  ```bash
  ./scripts/run_tests.sh
  ```

## بیلد

### ShadowNet-Inside (داخل ایران)

```bash
go build -tags inside -ldflags="-s -w" -o bin/shadownet-inside ./cmd/inside/
```

### ShadowNet-Outside (خارج از ایران)

```bash
go build -tags outside -ldflags="-s -w" -o bin/shadownet-outside ./cmd/outside/
```

## توزیع کانفیگ (خارج → داخل)

ShadowNet عمداً API مرکزی برای دریافت کانفیگ ندارد. بذرهای اولیه عمدتاً از راه‌های زیر می‌رسند:

1. پیام‌های ناپدیدشونده Signal با بسته‌های `snb://v2:` (امضاشده و رمزگذاری‌شده)
2. QR کد برای بوت‌استرپ حضوری
3. Bluetooth Mesh (داخل ↔ داخل) برای خاموشی اینترنت
